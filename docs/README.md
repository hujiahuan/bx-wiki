# BX Wiki · 内部文档中心

> 面向研发与运维的企业内网文档模板，基于 [Docsify](https://docsify.js.org/)，静态文件部署即可使用。

## 本站点包含什么

| 模块 | 说明 |
| --- | --- |
| 内部技术文档 | 架构、技术栈、安全与合规等「写给团队看」的背景材料 |
| 推荐平台（一期） | Go 推荐平台：边界与决策、总体架构、数据同步与身份归因、实施里程碑、工程规范（见 [推荐平台文档](recommend/README.md)） |
| 开发手册 | 环境、分支、评审等日常研发流程 |
| API 文档 | 接口约定、认证、错误码与示例 |
| 使用说明 | 面向业务方或内部用户的上手与 FAQ |
| 部署文档 | Nginx、子路径、HTTPS 等上线说明 |
| 运维手册 | 监控、日志、备份与故障 Runbook |

## 使用方式

- **本地预览**：仓库根目录执行 `npm run serve`，浏览器访问 `http://localhost:3000`。
- **服务器部署**：将 `docs/` 整目录作为网站根目录（或配置为静态站点根路径），确保 `index.html` 可被直接访问。详见 [部署总览](deploy/overview.md)。

## 维护约定（可改）

- 左侧目录在 `_sidebar.md` 维护；顶部导航在 `_navbar.md`。
- 全站检索依赖官方 `search` 插件，构建索引前请保证各 `.md` 路径可被访问。
- 若站点挂在子路径（如 `/wiki/`），必须在 `index.html` 的 `$docsify` 中设置 `basePath`。详见 [子路径部署](deploy/subpath.md)。
