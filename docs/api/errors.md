# 错误码与排障

## HTTP 状态码使用

| 状态码 | 含义 | 示例场景 |
| --- | --- | --- |
| 400 | 请求参数不合法 | JSON Schema 校验失败 |
| 401 | 未认证 | Token 缺失或过期 |
| 403 | 无权限 | RBAC 拒绝 |
| 404 | 资源不存在 | ID 无效 |
| 409 | 状态冲突 | 重复创建 |
| 429 | 限流 | 触发配额 |
| 500 | 服务内部错误 | 未捕获异常（应尽量避免暴露细节） |

## 业务错误体（示例）

```json
{
  "code": "ITEM_NOT_FOUND",
  "message": "资源不存在",
  "request_id": "a1b2c3d4"
}
```

## 排障清单

1. 记录 `request_id` / `trace_id`。
2. 对照 [监控与告警](../ops/monitoring.md) 查看对应时间段指标与日志。
3. 若为网关层错误，检查上游健康状态与超时配置。
