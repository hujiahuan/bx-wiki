# 故障应急 Runbook

## 通用流程

1. **确认影响面**：哪些区域 / 客户 / 功能不可用。
2. **止血**：限流、降级、回滚到上一稳定版本（按 [部署总览](../deploy/overview.md) 操作）。
3. **恢复验证**：核心用例自动化 + 手工抽检。
4. **复盘**：时间线、根因、改进项与跟踪 Owner。

## 场景模板

### 场景 1：依赖服务超时飙升

- 检查下游健康与配额；
- 临时调大超时 / 熔断阈值（评估副作用）；
- 与依赖方建立 war room。

### 场景 2：数据库主从延迟过大

- 暂停重任务 / 大查询；
- 评估只读流量是否可临时切主（风险高，需 DBA 审批）。

## 应急通讯录

填写 PagerDuty / 企业微信 on-call 群、升级路径（L1 → L2 → 管理层）。
