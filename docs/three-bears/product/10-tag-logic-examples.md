# 标签逻辑示例

> 来源：笨熊进货App_20260423说明用.xlsx - Sheet 10

## 一、统计类标签

### 近30天采购频次

- **标签名称**：近30天采购频次
- **标签逻辑说明**：统计商家用户在过去30天内的订单总数，反映其活跃程度
- **取数逻辑**：从 ODS 层 `order_fact` 表中筛选 `user_type = 'merchant'` 且 `order_status IN ('paid', 'delivered')` 的订单，按 `user_id` 聚合，统计过去30天内订单数量（按 `create_time` 过滤）

### 平均客单价

- **标签名称**：平均客单价
- **标签逻辑说明**：计算用户每次采购的平均金额，用于划分消费层级
- **取数逻辑**：基于 DWD 层 `dwd_order_item` 宽表，按 `user_id` 分组，计算 `SUM(gmv) / COUNT(order_id)`，剔除退款订单（`refund_status = 'none'`）

### 常购商品品类

- **标签名称**：常购商品品类
- **标签逻辑说明**：根据历史订单中出现频率最高的商品类目（如蔬菜、肉类、调味品）进行归类
- **取数逻辑**：从 DWD 层 `dwd_order_item` 关联 `dim_product` 商品维度表，按 `user_id` 和 `category_l1`（一级类目）统计购买次数，取 TOP1 作为主品类；若存在多个并列，则标记为"多品类混合"

## 二、规则类标签

### 高频采购用户

- **标签名称**：高频采购用户
- **标签定义**：近30天采购频次 ≥ 5，用于识别稳定活跃客户
- **取数逻辑**：直接复用"近30天采购频次"标签值，若 ≥ 5 则打标为"高频"，否则为"低频"；通过 DWS 层 `dws_user_behavior_summary` 每日更新

### 大额采购用户

- **标签名称**：大额采购用户
- **标签定义**：单次采购金额 ≥ 2000元 **或** 月累计采购额 ≥ 10000元
- **取数逻辑**：
  - 单次：从 `dwd_order_fact` 中筛选 `order_gmv >= 2000` 的订单，去重用户ID
  - 月累计：基于 `dws_user_monthly_summary` 中 `monthly_gmv >= 10000` 的用户打标
  - 两者取并集

### 促销敏感用户

- **标签名称**：促销敏感用户
- **标签定义**：用户超过60%的订单包含促销商品
- **取数逻辑**：关联 `dwd_order_item` 与 `dim_promotion`，统计每个用户在近90天内参与促销的订单数 / 总订单数 > 0.6，则打标"促销敏感"；需排除平台补贴类非用户感知促销

## 三、算法类标签

### 采购周期预测

- **标签名称**：采购周期预测
- **标签逻辑说明**：基于时间序列模型预测用户下次采购时间，提升推送精准度
- **取数逻辑**：使用 DWD 层 `dwd_order_fact` 中用户历史下单时间序列（按天聚合），采用 ARIMA 或 Prophet 模型训练，输出预测下次下单日期（每日更新预测值）

### 商品偏好推荐指数

- **标签名称**：商品偏好推荐指数
- **标签逻辑说明**：通过协同过滤算法分析用户历史行为，输出对各品类的偏好得分
- **取数逻辑**：基于 DWD 层用户-商品交互矩阵（`user_id × category_l1`），使用 ALS（交替最小二乘）算法训练推荐模型，输出每个用户对每个一级类目的偏好得分（0~1），保留 TOP3 品类

## 四、风险控制类标签

### 异常下单行为

- **标签名称**：异常下单行为
- **标签逻辑说明**：检测短时间内高频下单或退单行为，识别潜在刷单或操作异常风险
- **取数逻辑**：
  - 计算用户在1小时内下单数 > 5单
  - 或3天内退单率 > 40%（退单数/总订单数）
  - 触发"异常"标签
  - 数据源为 `dwd_order_fact` + `dwd_refund_fact`，由 Flink 实时流计算引擎监控
