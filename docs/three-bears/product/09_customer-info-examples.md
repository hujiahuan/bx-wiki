## 一、客户基础属性标签

### 需要获取的核心信息

| 信息类别 | 具体信息项 | 说明 |
|---------|-----------|------|
| 客户身份信息 | 客户ID、营业执照名称、统一社会信用代码 | 用于唯一标识客户身份，支持合规性校验与数据关联 |
| 经营主体信息 | 企业类型（个体户/公司）、法人代表、注册资本、成立日期 | 反映客户规模与经营稳定性，辅助信用评估与分层运营 |
| 经营类目信息 | 主营品类（如餐饮、零售、生鲜）、细分行业标签（如火锅店、便利店） | 用于商品推荐匹配、营销活动定向与供应链优化 |
| 地理位置信息 | 营业地址、所在城市/区县、配送半径 | 支持区域化选品、仓配调度与本地化运营策略制定 |
| 资质认证信息 | 食品经营许可证编号、卫生许可证、实名认证状态 | 确保平台合规准入，影响流量分发与服务权限 |
| 开店信息 | 入驻时间、店铺状态（营业/歇业/冻结）、是否连锁店 | 判断客户生命周期阶段，支持留存分析与唤醒策略 |
| 联系方式 | 联系人姓名、联系电话、紧急联系人 | 用于平台沟通、订单异常处理与客户服务触达 |

### 客户基础属性标签示例

| 标签名称 | 标签值示例 | 说明 |
|---------|-----------|------|
| 企业类型 | 个体工商户 / 有限责任公司 | 区分客户注册主体性质，影响信用评级与服务权限 |
| 主营品类 | 餐饮 > 火锅 / 零售 > 便利店 | 标注客户主要经营方向，用于商品推荐匹配与营销定向 |
| 所在城市 | 北京市 / 成都市 / 深圳市 | 支持区域化运营、本地仓配调度与差异化策略制定 |
| 入驻时间 | 注册日期时间戳 | 记录客户加入平台时间，用于生命周期分析与留存建模 |
| 店铺状态 | 营业中 / 已歇业 / 被冻结 | 反映当前可交易状态，影响流量分发与订单路由 |
| 实名认证状态 | 已认证 / 未认证 | 判断客户合规性，未认证客户可能限制部分功能使用 |
| 是否连锁店 | 是 / 否 | 识别连锁品牌客户，便于集中运营与批量服务对接 |
| 食品经营许可证状态 | 有效 / 过期 / 未上传 | 强合规性标签，直接影响客户是否可正常接单 |

## 二、动态行为标签

反映客户用户的行为特征、消费趋势和实时状态，具有**更新频率高、时效性强**的特点。

### 动态标签示例

| 标签类型 | 标签名称 | 标签逻辑说明 | 更新频率 |
|---------|---------|-------------|---------|
| 行为类标签 | 近7天登录频次 | 统计用户在过去7天内登录App的天数，衡量活跃度 | 每日更新 |
| 行为类标签 | 最近下单时间 | 记录用户最后一次成功下单的时间戳，用于判断流失风险 | 实时更新 |
| 消费类标签 | 月累计采购金额 | 统计当前自然月内用户的总采购GMV，用于消费层级划分 | 每日更新 |
| 消费类标签 | 本周客单价趋势 | 对比本周与上周平均客单价变化，识别消费升降级行为 | 每周更新 |
| 偏好类标签 | 近期偏好品类 | 基于过去30天订单中购买频次最高的商品类目，动态调整推荐策略 | 每日更新 |
| 风险类标签 | 异常登录行为 | 检测非常用设备、异地登录等操作，触发安全预警 | 实时流计算 |
| 营销类标签 | 当前促销参与状态 | 标记用户是否正在参与平台限时活动（如满减、秒杀） | 实时更新 |
| 服务类标签 | 当前未完成订单数 | 统计用户已支付但未完成履约的订单数量，影响服务优先级 | 实时更新 |
