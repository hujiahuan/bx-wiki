## 一、商品（SKU）画像系统

### 基础属性

- 品类、品牌、规格、保质期

### 业务属性

- 是否源头直采、是否季节性爆品、适用场景（如"适合烧烤店"）
- 关联商品ID（如买牛排必配黑椒酱）

### 热度指标

- 实时销量、区域销量排名、转化率、复购率

## 二、智能商品运营与分发体系

### A. 定向投放功能

允许运营人员设置规则，例如：
- "当用户所在区域=北京 且 商圈=写字楼，优先展示快餐盒饭类商品"

**新品/活动置顶：**
- 支持手动设置"新品优先"或"促销商品"在列表页的加权，确保运营活动能强曝光

**黑名单/去重机制：**
- 自动过滤缺货商品、该商户已拉黑的商品，或已购买过的周期性商品（避免重复推荐）

### B. 智能关联与周期计算模块

**复购计算器：**
- 系统需后台运行定时任务，根据用户历史订单计算"预计缺货时间"
- 在列表页触发"常购清单"或"补货提醒"接口

**关联规则挖掘：**
- 基于历史订单数据（如"买了A的人通常也买了B"），建立商品关联矩阵
- 当列表页加载时，实时插入这些互补品

### C. 排序与打分服务

这是"千人千面"的核心实现，采用"召回→排序→重排"的漏斗架构：

1. **多路召回**：同时从"热销榜"、"个性化偏好"、"复购池"、"新品池"中分别抓取一批商品
2. **加权排序模型**：给不同维度的商品打分，例如：
   - `总分 = 基础销量分 + (商圈匹配度 × 0.3) + (复购周期匹配度 × 0.5) + (价格优势 × 0.2)`
3. **打散与重排**：防止列表页连续出现10个同类商品，强制插入其他品类，保证浏览体验的丰富性

## 三、前端展示层

### 动态列表页

不再是固定的静态列表，而是根据接口返回的权重动态渲染。

### 推荐理由标签

在商品卡片上展示"推荐理由"，如：
- "您常买的品牌"
- "周边500家店主都在买"
- "预计您该补货了"

这能显著提升点击率。

### 场景化导购入口

在列表页顶部设置动态金刚区，根据季节自动切换（如夏天显示"烧烤食材专区"，冬天显示"火锅节"）。

## 四、评估与反馈系统

### A/B 测试平台

将商户分为两组，一组看"按销量排序"，一组看"按算法推荐排序"，对比两组的下单转化率。

### 埋点与反馈

记录商户对推荐商品的"点击"、"忽略"或"负反馈"（如点击"不感兴趣"），实时调整后续推荐结果。

## 五、商品标签体系（三张核心表）

### 标签定义表 (product_tag_define)

用于管理标签的"元数据"，定义标签的分类、类型和用途。

| 字段名 | 类型 | 说明 |
|-------|------|------|
| tag_id | BIGINT | 主键ID |
| tag_code | VARCHAR(50) | 标签唯一编码（系统用），如 `scene_hotpot` |
| tag_name | VARCHAR(100) | 标签显示名称，如 `适合火锅店` |
| tag_category | TINYINT | 标签分类：1-基础，2-业务，3-智能 |
| data_type | TINYINT | 数据类型：1-文本，2-数值，3-布尔 |
| source_type | TINYINT | 来源：1-人工录入，2-规则引擎，3-算法模型 |
| is_filterable | BOOLEAN | 是否支持搜索筛选 |
| status | TINYINT | 状态：1-启用，0-禁用 |

### 标签值表 (product_tag_value)

如果标签有固定选项（如"适用场景"下有"火锅"、"烧烤"），需要这张表来标准化选项。

| 字段名 | 类型 | 说明 |
|-------|------|------|
| value_id | BIGINT | 主键ID |
| tag_id | BIGINT | 关联标签定义ID |
| value_name | VARCHAR(100) | 标签值名称，如 `适合火锅店` |
| value_code | VARCHAR(50) | 标签值编码，如 `hotpot` |
| sort_order | INT | 排序权重 |

### 商品标签关联表 (product_tag_relation)

记录哪个商品拥有了哪个标签。

| 字段名 | 类型 | 说明 |
|-------|------|------|
| id | BIGINT | 主键ID |
| product_id | BIGINT | 商品ID |
| sku_id | BIGINT | SKU ID（可选，针对规格打标） |
| tag_id | BIGINT | 标签ID |
| tag_value_id | BIGINT | 标签值ID（关联具体选项） |
| tag_value_text | VARCHAR(100) | 标签值快照（冗余字段，防删库） |
| score | DECIMAL(5,2) | 标签置信度/权重（算法用） |
| expire_time | DATETIME | 过期时间（用于临时标签） |
| create_time | DATETIME | 创建时间 |

## 六、商品属性标签详解

### 1. 基础属性标签（静态）

描述商品"是什么"的最基本信息，通常来源于商品发布时的填写内容，相对稳定。

- **物理属性**：品类（如：生鲜/肉禽蛋）、品牌、产地、规格（如：5kg/箱）、材质、颜色、口味等
- **供应链属性**：供应商ID、采购成本、是否源头直采、所属仓库、配送时效等
- **合规属性**：生产许可证号、保质期、储存条件（如：-18℃冷冻）

### 2. 业务运营标签（动态/规则）

服务于具体的运营场景，用于商品的管理、筛选和营销，通常基于规则或动态数据生成。

- **销售状态**：新品、热销爆款、清仓商品、季节性商品（如：夏季限定）、平台主推
- **营销属性**：是否参与满减、有无优惠券、会员专享价、限时折扣
- **健康度指标**：高周转商品、滞销品、高毛利商品、低库存预警

### 3. 智能挖掘标签（预测/模型）

通过算法模型从海量数据中挖掘得出，能更深刻地揭示商品特征和关联关系。

- **内容理解标签**：利用图像识别技术分析商品主图，自动打上"简约风"、"适合送礼"、"网红款"等视觉或风格标签
- **关联关系标签**：基于用户共同购买行为，生成"常与牛排搭配"、"火锅伴侣"等关联标签，用于组合推荐
- **适用场景标签**：通过分析用户评价和搜索词，挖掘出"适合快餐店"、"宝宝辅食优选"等场景化标签
