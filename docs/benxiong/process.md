# 订单送货与退鸡蛋筐押金（业务版）

## 一句话说明

1. **送货**：货要先到仓库办好「签到」和「入库」，后台才能给司机派「送货单」；司机把货送到后拍照，订单才算**完成**。
2. **退筐押金**：只有**已付款且已完成**的订单，用户才能申请退鸡蛋筐等押金；系统会**尽量仍派当初送货的那位司机**上门取筐；若找不到这位司机，由**押金管理**里同事手动派司机。司机取走筐并拍照后，**财务**在后台操作，钱按原支付方式退回用户（微信或支付宝）。

## 流程图

在支持流程图语法的文档或编辑器中打开本文件即可预览图。

```mermaid
flowchart TB
    subgraph 第一步["一、用户下单"]
        甲[用户在手机端下单并完成付款]
    end

    subgraph 第二步["二、仓库这边（送货之前必须做完）"]
        乙{货物是否已经到仓库并完成签到？}
        乙 -->|否| 阻一[还不能给送货司机派单]
        乙 -->|是| 丙{是否已经在系统里完成入库？}
        丙 -->|否| 阻二[还不能给送货司机派单]
        丙 -->|是| 丁[订单进入「待派送货」队列，由同事在后台或仓库系统里处理]
    end

    subgraph 第三步["三、给送货司机派单"]
        丁 --> 戊[同事在后台选好送货司机]
        戊 --> 己{这位司机是否负责用户收货地址所在片区？}
        己 -->|否| 庚[提示：该司机不负责此区域，需换人]
        己 -->|是| 辛[生成送货任务并通知司机，订单进入配送中]
        辛 --> 壬[系统防止同一订单被重复派给多位司机]
    end

    subgraph 第四步["四、司机送货直到订单完成"]
        辛 --> 子[司机确认已从仓库取货出发]
        子 --> 丑[司机把货送到用户处，并拍摄送达现场照片上传]
        丑 --> 寅[订单在系统里标记为已完成]
    end

    subgraph 第五步["五、用户申请退鸡蛋筐押金"]
        寅 --> 卯{这笔订单里是否还有可退的筐类押金？}
        卯 -->|否| 辰[无可退金额或暂不符合申请条件]
        卯 -->|是| 巳[用户在手机端填写取筐地址等信息并提交申请]
        巳 --> 午[系统生成一张「押金回收单」，记下要退的筐数和金额]
        午 --> 未{能否找到当初送这笔货的那位司机？}
        未 -->|能| 申[状态：待司机上门取筐，并自动把任务派给这位司机]
        未 -->|不能| 酉[状态：待分配司机，需同事在「押金管理」里手动选司机]
        酉 --> 戌[同事选好司机后，任务发给司机，状态变为待上门取筐]
        申 --> 亥
        戌 --> 亥
    end

    subgraph 第六步["六、司机上门取鸡蛋筐（须拍照）"]
        亥[司机在手机端看到「取回押金筐」任务]
        亥 --> 子二[司机到达用户处，清点实际取回的筐数并拍摄取筐照片上传]
        子二 --> 丑二[押金单进入「货已取回，等待财务打款」状态]
    end

    subgraph 第七步["七、财务把钱退给用户"]
        丑二 --> 寅二[财务同事在「押金管理」中核对后发起退款]
        寅二 --> 卯二{用户当时是用什么方式付的款？}
        卯二 -->|微信| 辰二[通过微信原路退回押金]
        卯二 -->|支付宝| 巳二[通过支付宝原路退回押金]
        卯二 -->|其他| 午二[当前流程暂不支持自动退，需另行沟通处理]
        辰二 --> 未二[押金单标记为已退款，用户收到钱]
        巳二 --> 未二
        午二 --> 申二[若通道异常会进入需人工跟进的失败或待处理状态]
    end

    甲 --> 乙
    阻一 -.-> 乙
    阻二 -.-> 丙
```

## 泳道图与时间顺序对照（文字版）

若上图连线在个别阅读器中显示不理想，可按下面顺序向非技术同事口述：

| 顺序 | 谁 | 做什么 |
|:---:|:---|:---|
| 1 | 用户 | 下单并付款 |
| 2 | 仓库与仓管 | 货到签到 → 完成入库（两件事都做完才能派送货） |
| 3 | 后台同事 | 给订单指派送货司机（须匹配负责区域） |
| 4 | 司机 | 送货并在送达时拍照 |
| 5 | 用户 | 收货；订单在系统里变为「已完成」后，可申请退筐押金 |
| 6 | 系统优先 / 后台同事 | 尽量派「原送货司机」取筐；派不到则由押金管理里人工派司机 |
| 7 | 司机 | 上门取筐并拍照 |
| 8 | 财务 | 在押金管理里发起退款，钱原路退回用户微信或支付宝 |

---

## 押金单状态

- **待分配司机**：还没定下谁来取筐，需要后台在押金管理里选人。
- **待司机取回**：已通知司机，司机尚未完成取筐与拍照。
- **已取回待财务退款**：筐和照片已齐，等财务点退款。
- **已退款**：钱已退回用户。
- **退款失败 / 已取消**：按界面提示或联系运营、财务处理。
